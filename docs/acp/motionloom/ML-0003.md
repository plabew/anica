## 1) Hierarchy Overview

```xml
<Scene id="scene0">
  <Timeline>
    <Track id="..." space="..." z="...">
      <Sequence from="..." duration="..." out="...">
        <Layer id="..." zDepth="..." blend="...">
          <!-- visual nodes: Text, Rect, Circle, Group, Path, etc. -->
        </Layer>
      </Sequence>
    </Track>
  </Timeline>
</Scene>
```

## 2) `<Timeline>`

Root container for all time-based tracks within a `<Scene>`.

- Optional: `id`
- Must contain at least one `<Track>`

## 3) `<Track>`

A layer within the timeline. Multiple tracks are composited in `z` order.

- Required: `id`
- Optional:
  - `space` — `"world"` (default) or `"screen"` for content tracks only
  - `z` — draw order (higher = later/overlapping earlier)
  - `role` — `"camera"` for camera tracks, or omitted for content tracks
  - `zDepth` — camera-space depth for sorting (negative = closer)

### Camera Track

Camera tracks must use `role="camera"` and must not set `space`.

```xml
<Track id="camera" role="camera" z="1000">
  <Sequence from="0s" duration="3s" out="hold">
    <Camera x={curve("0:320:linear, 3:520:ease_in_out")}
            y="225"
            zoom={curve("0:1.00:linear, 3:1.35:ease_in_out")} />
  </Sequence>
</Track>
```

### Content Track

```xml
<Track id="main" space="world" z="0">
  <Sequence from="0s" duration="3s" out="hold">
    <Layer>
      <Circle x="320" y="225" radius="42" color="#9CA3AF" />
    </Layer>
  </Sequence>
</Track>
```

## 4) `<Sequence>`

A timed clip on a track. Defines start time and duration.

- Required: `duration`
- Optional:
  - `id` — unique identifier
  - `from` — start time relative to track start (e.g., `"0s"`, `"1.5s"`)
  - `out` — `"hold"` (default) or `"hide"`
    - `"hold"` — last frame persists after duration
    - `"hide"` — element disappears after duration

### Curve Reset

Curves inside a `<Sequence>` are **local to the sequence**. Time `0` in a curve equals the sequence start time.

```xml
<Sequence from="1.5s" duration="2s" out="hold">
  <Layer>
    <Rect x="0" y={curve("0:100:linear, 2:300:ease_out")} ... />
  </Layer>
</Sequence>
```
Above, `y` goes from 100 to 300 over the sequence duration (0 to 2 seconds, i.e., 1.5s to 3.5s globally).

## 5) `<Chain>`

A container for sequential `<Sequence>` children. Automatically schedules them back-to-back without hand-coded global offsets.

- Optional: `id`
- Optional: `from` — start time for the first child
- Optional: `gap` — gap between sequences (can be negative for overlap)

```xml
<Chain id="revenue_chain" from="1.15s" gap="-0.12s">
  <Sequence id="bar_q1" duration="0.85s" out="hold">
    <Layer>
      <Rect fill="url(#cyan_bar)" ... />
    </Layer>
  </Sequence>
  <Sequence id="bar_q2" duration="0.85s" out="hold">
    <Layer>
      <Rect fill="url(#amber_bar)" ... />
    </Layer>
  </Sequence>
</Chain>
```

- `bar_q1` runs from `1.15s` to `2.00s`
- `bar_q2` runs from `1.93s` to `2.78s` (0.85s - 0.12s gap)

## 6) `<Layer>`

Visual node container within a `<Sequence>`.

- Optional: `id`
- Optional: `zDepth` — camera-space depth sorting (negative = closer, positive = farther)
- Optional: `blend` — blend mode (e.g., `"screen"`, `"multiply"`, `"overlay"`)
- Optional: `opacity`

Multiple layers within one sequence are depth-sorted by `zDepth`:

```xml
<Sequence from="0s" duration="4s" out="hold">
  <Layer id="back_layer" zDepth="0.5">
    <Circle x="400" y="225" radius="120" color="#EA4335" />
  </Layer>
  <Layer id="front_layer" zDepth="-0.5">
    <Circle x="400" y="225" radius="80" color="#4285F4" />
  </Layer>
</Sequence>
```


## 8) Common Patterns

### Pattern A: Single track, single layer

```xml
<Scene id="simple">
  <Timeline>
    <Track id="main" space="world" z="0">
      <Sequence from="0s" duration="3s" out="hold">
        <Layer>
          <Text x="center" y="center" value="Hello" fontSize="48" color="#ffffff" />
        </Layer>
      </Sequence>
    </Track>
  </Timeline>
</Scene>
```

### Pattern B: Multiple layers with animation

```xml
<Scene id="animated">
  <Timeline>
    <Track id="main" space="world" z="0">
      <Sequence from="0s" duration="3s" out="hold">
        <Layer id="background">
          <Rect x="0" y="0" width="1920" height="1080" color="#000000" />
        </Layer>
        <Layer id="text">
          <Text x="center" y="center" value="Fade In"
                fontSize="96" color="#ffffff"
                opacity={curve("0:0:linear, 1:1:ease_out, 3:1:linear")} />
        </Layer>
      </Sequence>
    </Track>
  </Timeline>
</Scene>
```

### Pattern C: Chain of animated sequences

```xml
<Scene id="staggered">
  <Timeline>
    <Track id="dots" space="world" z="0">
      <Chain id="dot_chain" from="0s" gap="0.5s">
        <Sequence id="dot1" duration="1s" out="hold">
          <Layer>
            <Circle x="400" y="225" radius="0" color="#FF3B30"
                    opacity={curve("0:0:linear, 1:1:ease_out")} />
          </Layer>
        </Sequence>
        <Sequence id="dot2" duration="1s" out="hold">
          <Layer>
            <Circle x="480" y="225" radius="0" color="#FF9500"
                    opacity={curve("0:0:linear, 1:1:ease_out")} />
          </Layer>
        </Sequence>
      </Chain>
    </Track>
  </Timeline>
</Scene>
```

## ACP Reply (Short)

Always wrap visual nodes in `<Timeline><Track><Sequence><Layer>` for new scenes. Use `<Chain>` when you need sequential scheduling without hand-coded offsets. Use `zDepth` (negative = closer) for depth sorting. Use `out="hide"` for temporary elements that should disappear after their duration.
