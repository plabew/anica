Source of truth:
- `anica/crates/motionloom/src/dsl.rs`
- `anica/crates/motionloom/src/process/pass.rs`
- `anica/crates/motionloom/src/process/process_catalog.rs`

Use this document as the latest ACP-facing rule sheet before generating or explaining MotionLoom DSL.

## 1) Root Graph Rules

- A script must start with `<Graph ...>` after only whitespace or XML comments.
- `<Graph>` requires `fps` and `size`.
- `duration` is optional and defaults to `2s`.
- `renderSize` is optional and must have non-zero width and height when present.
- `scope` is removed; never generate it.
- `apply` is optional and defaults to `clip`; valid values are `clip` and `graph`.
- Tag names are case-sensitive; use `<Process>`, not `<process>`.

Recommended root form:

```xml
<Graph fps={60} duration="3s" size={[1920,1080]}>
  ...
  <Present from="scene0" />
</Graph>
```

## 2) Present Rules

- Exactly one `<Present ... />` is supported.
- `<Present>` must be a direct child of `<Graph>`.
- `<Present>` must be self-closing.
- `<Present>` must be the final node before `</Graph>`.
- `<Present>` requires `from`.
- `to` and `vsync` are optional.
- Do not put `<Present>` inside `<Scene>`, `<Process>`, `<Track>`, `<Layer>`, or any other child block.

Scene output:

```xml
<Present from="scene0" />
```

Process output:

```xml
<Process id="final_grade">
  ...
</Process>
<Present from="final_grade" />
```

## 3) Process Rules

`<Process>` is a valid graph-level container for process nodes. It aliases its internal output as a presentable graph resource.

- `<Process>` requires `id`.
- `<Process>` must contain nodes; it cannot be self-closing.
- `<Process>` may declare `output="resource_id"` or `present="resource_id"`.
- If no output is declared, output inference uses the last `<Output id="...">`, then last `<Pass out={...}>`, then last `<Tex id="...">`.
- Root `<Present from="...">` must reference the `<Process id>`, not the internal texture, when presenting the process container.
- Process parser supports a single `<Process>` block in direct process mode; use the root graph dispatcher for mixed graphs.

Recommended process form:

```xml
<Graph fps={60} duration="3s" size={[1920,1080]}>
  <Process id="final_grade">
    <Input id="clip0" type="video" from="input:clip0" />
    <Tex id="src" fmt="rgba16f" from="clip0" />
    <Tex id="out" fmt="rgba16f" size={[1920,1080]} />
    <Pass id="fx" effect="hsla_overlay" in={["src"]} out={["out"]}
          params={{ hue: "210", saturation: "0.7", lightness: "0.41", alpha: "0.45" }} />
  </Process>
  <Present from="final_grade" />
</Graph>
```

## 4) Pass Rules

`<Pass>` requires:
- `id`
- `effect`
- `in`
- `out`

`<Pass>` optional fields:
- `kind` defaults to `compute`.
- `kernel` is optional; when omitted, runtime resolves it from `effect_kernel_map.kv`.
- `mode`, `role`, `transition`, `transitionFallback`, `transitionEasing`, `transitionClips`, `params`, `iterate`, `pingpong`, `cache`, `blend`, `loadOp`, and `storeOp`.

Rules:
- Prefer omitting `kernel` when the effect exists in the kernel map.
- Use `params={{ key: "value" }}` for pass params.
- `params` entries are comma-separated `key: value` pairs.
- `kind="raster"` is not valid for current process passes.
- `effect` is required even when `kernel` is explicit.

## 8) Scene Rules

Recommended scene form:

```xml
<Scene id="scene0">
  <Timeline>
    <Track id="main" space="world" z="0">
      <Sequence from="0s" duration="3s" out="hold">
        <Layer>
          <Group id="title_group">
            <Text value="HELLO WORLD" x="center" y="center" fontSize="96" color="#ffffff" />
          </Group>
        </Layer>
      </Sequence>
    </Track>
  </Timeline>
</Scene>
```

Rules:
- `<Scene>` requires `id`.
- Prefer `<Timeline><Track><Sequence><Layer>` for generated scenes.
- `<Text>` requires `value`; do not use `text=`, `content=`, children text, or closing text bodies.
- `<Track role="camera">` only accepts `<Sequence>` children, and each camera sequence must contain exactly one self-closing `<Camera />`.
- `<Track role="camera">` must not set `space`; only content tracks use `space="world"` or `space="screen"`.
- A `<Camera>` outside a camera track is rejected.
- Visual content belongs in content tracks using `space="world"` or `space="screen"`.
- `z` controls track draw order; `zDepth` controls depth sorting where negative is closer.
- Every `url(#id)` paint reference must have a matching `<LinearGradient id="id" ... />` or `<RadialGradient id="id" ... />` in `<Defs>` before using it.
- Direct scene nodes are still accepted for backwards compatibility, but ACP should generate timeline scenes.

## 9) Scene + Process Composition

Use `Scene -> Tex -> Pass -> Present` when applying post-process effects to generated scene content:

```xml
<Graph fps={60} duration="3s" size={[1920,1080]}>
  <Scene id="scene0">
    <Timeline>
      <Track id="main" space="world" z="0">
        <Sequence from="0s" duration="3s" out="hold">
          <Layer>
            <Text value="Blur Demo" x="center" y="center" fontSize="96" color="#ffffff" />
          </Layer>
        </Sequence>
      </Track>
    </Timeline>
  </Scene>

  <Tex id="scene_tex" fmt="rgba16f" from="scene:scene0" />
  <Tex id="blur_h" fmt="rgba16f" size={[1920,1080]} />
  <Tex id="blur_out" fmt="rgba16f" size={[1920,1080]} />
  <Pass id="fx_blur_h" effect="gaussian_5tap_h" in={["scene_tex"]} out={["blur_h"]} params={{ sigma: "8.0" }} />
  <Pass id="fx_blur_v" effect="gaussian_5tap_v" in={["blur_h"]} out={["blur_out"]} params={{ sigma: "8.0" }} />
  <Present from="blur_out" />
</Graph>
```

## 10) Hard Do-Not-Generate List

- `<Graph scope="...">`
- `<Solid ... />`
- `<process ...>` lowercase
- nested `<Present>`
- more than one `<Present>`
- `<Present>` before other graph nodes
- `<Pass kind="raster" ...>`
- `<Pass>` without `effect`
- `<Pass>` without `in` or `out`
- `<Text text="...">`, `<Text content="...">`, or `<Text>...</Text>`
- `<Camera>` as a visual container in a normal content track

## ACP Short Reply

Use `<Graph fps={...} size={[w,h]}>` with one final direct-child `<Present />`. Generate scenes as `<Scene><Timeline><Track><Sequence><Layer>...`, route post-process with `Tex from="scene:id"` and `<Pass effect=... in=... out=...>`, and omit `kernel` when the effect is mapped. Never generate `scope`, lowercase `<process>`, nested/multiple `<Present>`, raster passes, or text bodies.
