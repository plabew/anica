# ML-0006: MotionLoom Error Repair Guide and Golden Templates

Source of truth:
- `anica/crates/motionloom/src/dsl.rs`

- `anica/crates/motionloom/src/scene/render.rs`

Use this document when MotionLoom generation fails validation, parse, compile, or render, or when an LLM needs a safe starting template.

## 1) Repair Workflow

When a MotionLoom tool call fails:

1. Read the exact error text.
2. Classify it using the table below.
3. Fix the smallest invalid DSL pattern.
4. Call `anica.motionloom/set_scene_script` again with `apply_now=true`.
5. Only call `anica.motionloom/render_scene` after `set_scene_script` succeeds.

Do not replace the whole script unless the structure is fundamentally wrong. Prefer local edits that preserve user intent.

## 2) Common Error Repairs

| Error fragment | Root cause | Repair |
|---|---|---|
| `gradient reference not found: bg_glow` | A paint uses `url(#bg_glow)` but no matching gradient exists. | Add `<Defs><LinearGradient id="bg_glow" ... /></Defs>` before visual nodes, or replace the paint with a solid color. |
| `invalid scene paint 'url(#...)'` | Missing gradient, malformed color, or unsupported paint expression. | Use `#rrggbb`, `#rrggbbaa`, or define the referenced gradient in `<Defs>`. |
| `<Scene> root only accepts <Defs> and <Timeline>` | Visual nodes were placed directly inside `<Scene>`. | Wrap visual nodes as `<Scene><Timeline><Track><Sequence><Layer>...`. |
| `<Track role="camera"> must not set space` | A camera track used content-track `space`. | Remove `space` from the camera track; keep `space="world"` or `space="screen"` only on content tracks. |
| `expects MotionLoom Graph DSL` | Output is not a `<Graph ...>...</Graph>` script. | Return only a complete `<Graph>` block, no prose around tool arguments. |
| `parse error at line ...` | XML-like tag shape, required field, or brace syntax is invalid. | Fix the exact line first; do not invent unsupported tags. |
| `unsupported kernel(s)` | Pass references a kernel not embedded in the runtime. | Omit `kernel` and use a mapped `effect`, or choose a known effect from `ML-0005`. |
| `Pass ... without effect` | Current pass runtime requires `effect`. | Add `effect="..."` even when `kernel` is explicit. |
| `kind="raster"` | Raster pass kind is not supported. | Remove `kind` or use the default compute pass form. |
| `<Text text=...>` or `<Text>...</Text>` | Scene text content must use `value`. | Use `<Text value="..." ... />`. |
| `Present from="scene:scene0"` | `scene:` prefix is only for `Tex from`, not direct presentation. | Use `<Present from="scene0" />`; use `<Tex from="scene:scene0" />` only for post-process. |
| more than one `<Present>` | Only one final direct-child present node is supported. | Keep one `<Present ... />` as the final graph child. |
| nested `<Present>` | `<Present>` is inside a container. | Move it to the graph root as the final child. |

## 3) Golden Template A: Minimal Timeline Scene

Use this when the user asks for a simple generated scene.

```xml
<Graph fps={60} duration="3s" size={[1920,1080]}>
  <Background color="#070b14" />
  <Scene id="scene0">
    <Timeline>
      <Track id="main" space="world" z="0">
        <Sequence from="0s" duration="3s" out="hold">
          <Layer id="main_layer">
            <Group id="title_group" x="960" y="540" opacity="1">
              <Text value="HELLO WORLD"
                    x="0"
                    y="0"
                    fontSize="96"
                    color="#ffffff"
                    opacity={curve("0:0:ease_in_out, 0.6:1:linear, 3:1:linear")} />
            </Group>
          </Layer>
        </Sequence>
      </Track>
    </Timeline>
  </Scene>
  <Present from="scene0" />
</Graph>
```

## 4) Golden Template B: Scene With Safe Gradients

Use this when a design needs gradient backgrounds, glowing panels, or bar/shape gradients. Every `url(#id)` has a matching `<Defs>` entry.

```xml
<Graph fps={60} duration="4s" size={[1920,1080]}>
  <Background color="#050814" />
  <Scene id="scene0">
    <Defs>
      <LinearGradient id="bg_glow" x1="0" y1="0" x2="1" y2="1"
                      stops="0:#102347, 0.55:#08111f, 1:#050814" />
      <LinearGradient id="panel_glow" x1="0" y1="0" x2="0" y2="1"
                      stops="0:#234a7aee, 1:#0d1930ee" />
      <LinearGradient id="accent_bar" x1="0" y1="0" x2="0" y2="1"
                      stops="0:#54d6ff, 1:#2668ff" />
    </Defs>
    <Timeline>
      <Track id="main" space="world" z="0">
        <Sequence from="0s" duration="4s" out="hold">
          <Layer id="background_layer">
            <Rect x="0" y="0" width="1920" height="1080" fill="url(#bg_glow)" />
            <Rect x="320" y="220" width="1280" height="620" radius="36"
                  fill="url(#panel_glow)" stroke="#5e8fd8" strokeWidth="2" />
          </Layer>
          <Layer id="content_layer">
            <Group id="chart_group" x="460" y="760" opacity="1">
              <Rect x="0"
                    y={curve("0:240:ease_in_out, 1.2:60:linear, 4:60:linear")}
                    width="180"
                    height={curve("0:0:ease_in_out, 1.2:180:linear, 4:180:linear")}
                    fill="url(#accent_bar)" />
              <Text value="Revenue Lift" x="300" y="-420" fontSize="72" color="#ffffff" />
            </Group>
          </Layer>
        </Sequence>
      </Track>
    </Timeline>
  </Scene>
  <Present from="scene0" />
</Graph>
```

## 5) Golden Template C: Scene + Post-Process Blur

Use this when the user wants a generated scene and GPU post-process.

```xml
<Graph fps={60} duration="3s" size={[1920,1080]}>
  <Scene id="scene0">
    <Timeline>
      <Track id="main" space="world" z="0">
        <Sequence from="0s" duration="3s" out="hold">
          <Layer id="main_layer">
            <Rect x="0" y="0" width="1920" height="1080" fill="#101828" />
            <Text value="SOFT FOCUS"
                  x="620"
                  y="500"
                  fontSize="110"
                  color="#ffffff"
                  opacity="1" />
          </Layer>
        </Sequence>
      </Track>
    </Timeline>
  </Scene>
  <Tex id="scene_tex" fmt="rgba16f" from="scene:scene0" />
  <Tex id="blur_h" fmt="rgba16f" size={[1920,1080]} />
  <Tex id="blur_out" fmt="rgba16f" size={[1920,1080]} />
  <Pass id="blur_horizontal"
        effect="gaussian_5tap_h"
        in={["scene_tex"]}
        out={["blur_h"]}
        params={{ sigma: "4.0" }} />
  <Pass id="blur_vertical"
        effect="gaussian_5tap_v"
        in={["blur_h"]}
        out={["blur_out"]}
        params={{ sigma: "4.0" }} />
  <Present from="blur_out" />
</Graph>
```

## 7) Golden Template E: Repairing Missing Gradient

Broken:

```xml
<Rect x="0" y="0" width="1920" height="1080" fill="url(#bg_glow)" />
```

Fixed with a gradient:

```xml
<Defs>
  <LinearGradient id="bg_glow" x1="0" y1="0" x2="1" y2="1"
                  stops="0:#112244, 1:#050814" />
</Defs>
<Rect x="0" y="0" width="1920" height="1080" fill="url(#bg_glow)" />
```

Fixed with a solid fallback:

```xml
<Rect x="0" y="0" width="1920" height="1080" fill="#050814" />
```

Prefer the gradient fix when the user requested visual richness. Prefer the solid fallback when the user requested speed or a minimal repair.

## 8) Safe Generation Checklist

Before calling `set_scene_script`, verify:

- The script is one complete `<Graph ...>...</Graph>`.
- `fps={60}` and `size={[w,h]}` use brace syntax.
- There is exactly one root-level final `<Present ... />`.
- `Present from="scene0"` references the scene id directly.
- `Tex from="scene:scene0"` uses the `scene:` prefix only for post-process.
- `<Scene>` contains only `<Defs>` and `<Timeline>` as direct children.
- Visual nodes are inside `<Timeline><Track><Sequence><Layer>`.
- Camera tracks use `role="camera"` and do not set `space`.
- Every semantic group has an `id`.
- `<Text>` uses `value`, not `text`, `content`, or body text.
- Every `url(#id)` paint has a matching gradient in `<Defs>`.
- Every `<Pass>` has `id`, `effect`, `in`, and `out`.
- No unsupported tags are present: `<Solid>`, `<Animate>`, nested `<Present>`, or `<Pass kind="raster">`.

## ACP Short Reply

For failed MotionLoom generation, classify the error, apply the smallest repair, then retry `anica.motionloom/set_scene_script` before rendering. Start from the golden templates here when creating new scripts, especially timeline scenes and gradient scenes.
