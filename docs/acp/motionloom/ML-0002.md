# ML-0002: Natural-Language Scene Prompt -> Valid MotionLoom Scene DSL

Source of truth:
- `anica/crates/motionloom/src/dsl.rs`
- `anica/src/ui/motionloom_page.rs`
- `anica/src/api/motionloom.rs`
- `anica/src/api/transport_acp.rs`

## Problem This Doc Solves

When user says natural language like:
- "Create a 3-second black background with centered white HELLO WORLD fade-in"

ACP must output **valid MotionLoom scene DSL**.
Do not output legacy pass syntax that parser rejects for this use case.

## Mandatory Scene DSL Shape

For this kind of prompt, prefer:
1. `<Graph scope="scene" ...>`
2. `<Scene id="scene0">...</Scene>`
3. Scene children (`<Solid>`, `<Text>`, `<Image>`, `<Svg>`, `<Group>`, `<Mask>`, `<Character>`, `<Camera mode="2d">`, `<Rect>`, `<Circle>`, `<Line>`, `<Polyline>`, `<Path>`, `<Shadow>`)
4. `<Present from="scene0" />`

Minimal valid template:

```jsx
<Graph scope="scene" fps={60} duration="3s" size={[1920,1080]}>
  <Scene id="scene0">
    <Solid color="#000000" />
    <Text value="HELLO WORLD"
          x="center"
          y="center"
          fontSize="96"
          color="#ffffff"
          opacity="min($time.sec/1.0,1.0)" />
  </Scene>
  <Present from="scene0" />
</Graph>
```

Direct scene nodes without `<Scene>` are still accepted for backwards compatibility:
`<Solid>`, `<Text>`, `<Image>`, `<Svg>`, then `<Present from="scene" />`.

Use primitives for commercial UI/motion-graphics:

```jsx
<Graph scope="scene" fps={60} duration="6s" size={[1920,1080]}>
  <Scene id="scene0">
    <Solid color="[0.96,0.96,0.96,1]" />
    <Group x="120" y="110" rotation="-5" scale="1" opacity="1">
      <Shadow x="0" y="18" blur="32" color="[0,0,0,0.18]" />
      <Rect width="360" height="84" radius="22" color="[1,1,1,1]" stroke="[0.85,0.85,0.85,1]" strokeWidth="1" />
      <Circle x="42" y="42" radius="7" color="[0,0.32,0.95,1]" />
      <Text value="TechCrunch - OpenAI..." x="66" y="24" width="260" fontSize="28" lineHeight="34" color="[0.08,0.08,0.08,1]" />
    </Group>
  </Scene>
  <Present from="scene0" />
</Graph>
```

Use `<Character>` for dense vector character/face/hair scenes. Keep high-volume sketch strokes inside the character container:

```jsx
<Character x="960" y="540" scale="1.1" rotation="-2">
  <Path d="M -230 -150 C -120 -300 120 -300 230 -150 C 170 -30 -170 -30 -230 -150 Z"
        fill="#f8c7d6"
        stroke="#d85a78"
        strokeWidth="8" />
  <Mask shape="circle" x="0" y="45" radius="220">
    <Path d="M -140 -180 C -120 -60 -90 20 -40 110"
          stroke="#5d5961"
          strokeWidth="8"
          lineCap="round"
          lineJoin="round"
          taperStart="0.2"
          taperEnd="0.25" />
  </Mask>
  <Circle x="-70" y="70" radius="28" color="#e05b78" />
</Character>
```

For post-process, keep scene composition separate from GPU/compute passes:

```jsx
<Graph scope="scene" fps={60} duration="3s" size={[1920,1080]}>
  <Scene id="scene0">
    <Solid color="#000000" />
    <Text value="HELLO WORLD" x="center" y="center" fontSize="96" color="#ffffff" opacity="1" />
  </Scene>
  <Tex id="src" fmt="rgba16f" from="scene:scene0" />
  <Tex id="out" fmt="rgba16f" size={[1920,1080]} />
  <Pass id="fx_opacity" kind="compute" effect="opacity" in={["src"]} out={["out"]} params={{ opacity: "0.8" }} />
  <Present from="out" />
</Graph>
```

Color values may be HEX or BGRA arrays:
- `color="#ffffff"`
- `color="[1,1,1,1]"` for normalized BGRA
- `color="[255,255,255,255]"` for byte-range BGRA

Fade should be written in node expression (`opacity="$time..."`), not in a separate animate node.

## Do Not Generate (Common Failure Patterns)

For simple scene text overlays, do not use:
- `<Pass ... kind="raster" ...>`
- `effect="text_overlay"`
- any non-supported pass-kind value (parser only accepts `compute` or `render` in `<Pass>`)

If you need text in a scene, prefer `<Text ... />` directly.
MotionLoom text content is always `value="..."`.

Also do not emit:
- `<Animate ... />` (not supported by scene parser/runtime)
- `duration_ms` on `<Present ...>` as the only duration source (use `<Graph duration="...">`)
- `<Text text="...">`, `<Text content="...">`, `<Text children="...">`, or `<Text>...</Text>`

## ACP Tool Workflow For Natural-Language VFX

1. Call `anica.motionloom/set_scene_script`
   - pass generated DSL string
   - set `apply_now=true`
2. If user asked for output video / render:
   - call `anica.motionloom/render_scene`
   - default `mode="gpu"` on macOS
   - use `mode="gpu_prores"` or `mode="compatibility_cpu"` only when explicitly requested

## Example Tool Calls

Set + apply:

```json
{
  "script": "<Graph scope=\"scene\" fps={60} duration=\"3s\" size={[1920,1080]}> ... </Graph>",
  "apply_now": true,
  "focus_vfx_page": true
}
```

Render (GPU):

```json
{
  "mode": "gpu",
  "focus_vfx_page": true
}
```
