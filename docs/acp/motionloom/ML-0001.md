## 1) Scope Mapping (3 Modes)

MotionLoom user intent maps to these scopes:

1. `layer` (Layer Effect)
   - Affect clips underneath on timeline layer span.
2. `clip` (Per-Clip VFX)
   - Affect one selected clip only.
3. `scene` (Mini VFX Scene)
   - Generate/compose scene result from graph resources.

Note:
- Scope controls *where* effect applies.
- Process category controls *what* effect does.

## 3) DSL Tags (7 Core Tags)

1. `<Graph ...>`
2. `<Input ... />`
3. `<Tex ... />`
4. `<Buffer ... />`
5. `<Pass ... />`
6. `<Output ... />`
7. `<Present ... />`

## 4) Explicit vs Implicit Parameters

Rule:
- Required fields must be explicit.
- Optional fields can be omitted and rely on defaults.
- `effect` is now required on every `<Pass ... />` (all 7 process categories use the same mental model).

### `<Graph>`
- Required:
  - `scope` (`layer | clip | scene`)
  - `fps`
  - `size`
- Optional:
  - `id`
  - `version`
  - `apply` (default: `clip`)
  - `duration` (default: `2s` / `2000ms`)

`apply` behavior:
- `apply="clip"`: full clip span (default)
- `apply="graph"`: effect active only within `duration`

### `<Pass>`
- Required:
  - `id`
  - `effect`
  - `in`
  - `out`
- Optional:
  - `kernel` (if omitted, runtime resolves via `effect_kernel_map.kv`)
  - `kind` (default: `compute`)
  - `role` (`effect | transition`)
  - `transition`
  - `transitionFallback`
  - `transitionEasing`
  - `transitionClips`
  - `params`
  - `iterate`
  - `pingpong`
  - `cache`
  - `blend`
  - `loadOp`
  - `storeOp`

`blur_sharpen_detail_gaussian.wgsl` pass effect values:
- `gaussian_5tap_h`
- `gaussian_5tap_v`
- `box`
- `unsharp`

`transition_core.wgsl` pass effect values:
- `fade_in`
- `fade_out`
- `dip`
- `dissolve`

Transition timing guidance:
- Keep Graph at default `apply="clip"` (or omit `apply`).
- Do not depend on `Graph.duration` for transition timing.
- Put timing in `params`, for example:
  - `startSec`
  - `durationSec`
  - or explicit `progress` expression
  - optional `easing` in `params`
- `fade_out` special default: if `startSec` is omitted, it auto-anchors to clip tail (`clipDuration - durationSec`).

### Other nodes
- `<Input>` required: `id`
- `<Tex>` required: `id`, `fmt`
- `<Buffer>` required: `id`, `elemType`
- `<Output>` required: `id`, `to`
- `<Present>` required: `from`

## 5) Duration Format Guidance

Supported examples:
- `duration="2s"`
- `duration="2.35s"`
- `duration="2350ms"`

If omitted:
- defaults to `2s`.

## 6) Recommended Effect Authoring Process (for ACP Guidance)

When user asks "how to write this effect", answer with this order:

1. Pick scope:
   - layer / clip / scene
2. Pick one of 10 process categories.
3. Start with minimal graph (`Input -> Tex -> Pass -> Present`).
4. Use one kernel first (single pass), then add params.
5. Run and inspect runtime status:
   - `supported/total passes`
6. If needed, add second pass or branching graph.
7. Only then add advanced fields (`iterate`, `pingpong`, `cache`, blend/load/store).

## 7) Copy-Paste Templates

### A) Minimal full-clip blur (default `apply=clip`)
```jsx
<Graph scope="clip" fps={60} size={[1920,1080]}>
  <Input id="clip0" type="video" from="input:clip0" />
  <Tex id="src" fmt="rgba16f" from="clip0" />
  <Tex id="out" fmt="rgba16f" size={[1920,1080]} />
  <Pass id="fx_blur" kind="compute"
        effect="gaussian_5tap_h"
        in={["src"]} out={["out"]}
        params={{ sigma: "10.0" }} />
  <Present from="out" />
</Graph>
```

### B) Blur only in first 2 seconds (`apply=graph`)
```jsx
<Graph scope="clip" fps={60} apply="graph" duration="2s" size={[1920,1080]}>
  <Input id="clip0" type="video" from="input:clip0" />
  <Tex id="src" fmt="rgba16f" from="clip0" />
  <Tex id="out" fmt="rgba16f" size={[1920,1080]} />
  <Pass id="fx_blur" kind="compute"
        effect="gaussian_5tap_h"
        in={["src"]} out={["out"]}
        params={{ sigma: "10.0" }} />
  <Present from="out" />
</Graph>
```

### C) Testing preset kernel
```jsx
<Pass id="fx_blur" kind="compute" kernel="effect_for_testing_run.wgsl" effect="gaussian_5tap_h"
      in={["src"]} out={["out"]}
      params={{ sigma: "10.0" }} />
```

## 8) Removed Legacy Kernel Names (Do Not Recommend)

These are removed from current runtime mapping and should not be suggested:
- `layer_adjust_keyframe.wgsl`
- `zoom_pan_keyframe.wgsl`
- `transition_fade_in_out.wgsl`
- `transition_dissolve.wgsl`

## ACP Reply (Short)
Use `scope` for where the effect applies (`layer|clip|scene`), and use the 10 process categories for effect type. Keep required fields explicit (`Graph: scope+fps+size`, `Pass: id+effect+in+out`) and omit optional fields for a clean script. `kernel` may be omitted when effect mapping exists. Default behavior is `apply="clip"` with default `duration="2s"` when omitted.
