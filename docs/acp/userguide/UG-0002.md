# UG-0002: Full Setup Guide — Build Anica and Connect ACP

## Scope

This guide covers the complete setup from zero to a working Anica editor with ACP chat connected. It includes installing Rust, system dependencies, building Anica, and setting up AI CLI tools.

Target audience: first-time users who want to build from source and use AI features.

---

## Step 1: Install Rust

```bash
curl --proto '=https' --tlsv1.2 -sSf https://sh.rustup.rs | sh
```

Restart your terminal after install.

```bash
rustc --version
cargo --version
```

---

## Step 2: Install system dependencies

### macOS

```bash
xcode-select --install

brew install \
  gstreamer \
  gst-plugins-base \
  gst-plugins-good \
  gst-editing-services \
  ffmpeg
```

### Ubuntu / Debian

```bash
sudo apt-get update
sudo apt-get install -y \
  build-essential pkg-config cmake \
  libgstreamer1.0-dev \
  libgstreamer-plugins-base1.0-dev \
  gstreamer1.0-plugins-base \
  gstreamer1.0-plugins-good \
  gstreamer1.0-editing-services \
  libges-1.0-dev \
  ffmpeg \
  libxkbcommon-dev \
  libwayland-dev \
  libvulkan-dev
```

### Windows

1. Install [Visual Studio Build Tools](https://visualstudio.microsoft.com/visual-cpp-build-tools/) — select "Desktop development with C++".
2. Install [GStreamer MSVC 64-bit](https://gstreamer.freedesktop.org/download/) — both runtime and development.
3. Install [FFmpeg](https://www.gyan.dev/ffmpeg/builds/) — add `bin/` to `PATH`.

---

## Step 3: Clone and build Anica

```bash
git clone https://github.com/LOVELYZOMBIEYHO/anica.git
cd anica
cargo build
```

This also compiles `anica-acp` (the ACP agent binary) automatically.

Run:

```bash
cargo run
```

---

## Step 4: Install an AI CLI (pick one or more)

All three require Node.js. Install it first if needed:

**macOS:** `brew install node`
**Linux:** `curl -fsSL https://deb.nodesource.com/setup_lts.x | sudo -E bash - && sudo apt-get install -y nodejs`
**Windows:** Download from [nodejs.org](https://nodejs.org/)

### Codex (OpenAI)

```bash
npm install -g @openai/codex
codex login
```

If browser is not available:

```bash
codex login --device-auth
```

### Gemini (Google)

```bash
npm install -g @google/gemini-cli
gemini
# Type /auth inside the Gemini CLI
# Select oauth-personal, follow browser flow
```

Or set an API key:

```bash
export GEMINI_API_KEY="your-key-here"
```

### Claude (Anthropic)

```bash
npm install -g @anthropic-ai/claude-code
claude auth login
```

Check status:

```bash
claude auth status
```

---

## Step 5: Connect ACP in Anica

1. Open Anica (`cargo run`).
2. Click the **AI Agents** icon in the left sidebar.
3. Select your provider: **Codex**, **Gemini**, or **Claude**.
4. The **ACP Agent Command** field should show the path to `anica-acp`. If blank, set it to:
   ```
   /path/to/anica/target/debug/anica-acp
   ```
5. Click **Connect**.
6. Green status text confirms connection. The AI Chat widget (bottom-right) is now usable.

---

## Step 6: Test ACP

Open the AI Chat widget and try:

```
What clips are on my timeline?
```

The agent should respond with your current timeline structure.

Other example prompts:

| Prompt | What it does |
|--------|-------------|
| `Cut all silence below -40dB` | Detects and removes silent sections |
| `Translate S1 subtitle to English` | Creates a new subtitle track with translations |
| `Export to MP4` | Renders the timeline to an MP4 file |

## Troubleshooting

| Symptom | Fix |
|---------|-----|
| `cargo build` fails on GStreamer | Reinstall GStreamer dev packages (step 2) |
| Linker errors on macOS | `xcode-select --install` |
| ACP says "not found" | Check `anica-acp` path in ACP Agent Command field |
| `codex login` hangs | Try `codex login --device-auth` instead |
| Gemini says "no credentials" | Run `gemini` → `/auth` → `oauth-personal` |
| Claude auth fails | Run `claude auth login` and follow prompts |
| Video preview is black | Check GStreamer plugins are installed (`gst-plugins-base` + `gst-plugins-good`) |
