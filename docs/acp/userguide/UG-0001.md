# UG-0001: NPM Install Methods + Codex/Gemini/Claude CLI Quick Start

## Scope
This guide covers:
- How to install `npm` (Node.js package manager).
- `npm i -g @openai/codex`
- `codex login`
- `codex login --device-auth`
- `npm install -g @google/gemini-cli`
- Gemini CLI login without API key (`gemini`, then `/auth`)
- Claude CLI install/login (`claude auth login`, `claude auth status`)

## 1) Install Node.js and npm

`npm` is included with Node.js. If `npm` is not available, install Node.js first.

### Option A: nvm (recommended for managing versions)
```bash
nvm install --lts
nvm use --lts
node -v
npm -v
```

### Option B: Homebrew (macOS)
```bash
brew install node
node -v
npm -v
```

If `node -v` and `npm -v` both return versions, npm is ready.

## 2) Install OpenAI Codex CLI (global)
```bash
npm i -g @openai/codex
```

`npm i` is shorthand for `npm install`, so this is equivalent to:
```bash
npm install -g @openai/codex
```

Verify:
```bash
codex --version
codex --help
```

## 3) Login to Codex CLI

Default login flow:
```bash
codex login
```

Force device-auth flow:
```bash
codex login --device-auth
```

`--device-auth` is a flag parameter (no extra value needed).  
Use it when browser-based login is not convenient (for example remote shell/SSH environments).

## 4) Install Google Gemini CLI (global)
```bash
npm install -g @google/gemini-cli
```

Verify:
```bash
gemini --version
gemini --help
```

## 5) Gemini CLI Login (Without API Key)
```bash
gemini
```

Inside Gemini CLI, run:
```text
/auth
```

Then select a login method (for example, **Login with Google**) and complete the browser flow.

For ACP mode in Anica:
- Codex, Gemini, and Claude all run through bundled `anica-acp` (same tool-bridge flow).
- API key is optional for Gemini.
- If Gemini CLI login is already completed, ACP can use that CLI auth state.
- If needed, you can still provide `GEMINI_API_KEY` and it will be forwarded on connect.

## 6) Install Claude CLI and Login

Recommended installer (official):
```bash
curl -fsSL https://claude.ai/install.sh | bash
```

Alternative npm path (legacy):
```bash
npm install -g @anthropic-ai/claude-code
```

Login and verify:
```bash
claude auth login
claude auth status
claude --version
```

For ACP mode in Anica:
- Choose `Claude` in ACP Provider.
- Keep `ACP Agent Command` as bundled `anica-acp`.
- Claude CLI login is supported; `ANTHROPIC_API_KEY` is optional.

## 7) Quick Troubleshooting
- If global install fails with permissions, prefer `nvm` Node installs.
- After installation, restart terminal or reload shell config if command is not found.
- Re-check PATH with `which node`, `which npm`, `which codex`, `which gemini`, and `which claude`.
