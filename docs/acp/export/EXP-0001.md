# Anica Export Choices

Source of truth: `anica/src/core/export.rs` (`ExportPreset`, `ExportSettings`).

## Export Modes

1. `smart_universal` (recommended)
   - Try `Keep Source (Copy)` first.
   - Auto-fallback to preset re-encode when timeline/output requires rendering.
2. `keep_source_copy`
   - Stream copy trim only (`-c copy`).
   - Works only for simple single-source timeline with no effects/overlays/mix.
3. `preset_reencode`
   - Always use selected preset encoder pipeline (same as previous exporter behavior).

## ExportSettings Normalization

- `fps`: clamp `1..=144`
- `crf`: clamp `0..=51`
- `audio_bitrate_kbps`: clamp `64..=512`
- `encoder_preset`: allowed
  - `ultrafast`, `superfast`, `veryfast`, `faster`, `fast`, `medium`, `slow`, `slower`, `veryslow`
  - fallback: `medium`

## Recommendation Guideline (for ACP Agent)

When user asks "export same quality/codec as source":

1. Call `anica.media_pool/list_metadata` first.
2. Read source fields: `video_codec`, `audio_codec`, `fps`, `width`, `height`, `media_kind`.
3. Map to closest Anica preset:
   - Source H.264 8-bit -> `h264_mp4`
   - Source HEVC/H.265 10-bit -> `hevc_mp4` or `x265_main10_mp4`
   - Source VP9 -> `vp9_webm`
   - Source ProRes 422/HQ -> `prores_422_mov` / `prores_422_hq_mov`
   - Source ProRes 444/4444 -> `prores_444_mov` / `prores_4444_mov`
   - Audio-only source -> choose from audio-only presets.
4. If exact codec/profile is unavailable, provide "closest visually similar" option and a "max compatibility" fallback (`h264_mp4`).
5. Be explicit about tradeoff: compatibility vs file size vs edit-master quality.

## Notes

- CRF-based presets are generally distribution outputs.
- DNxHR/ProRes are mezzanine/edit-master outputs.
- For strict alpha workflows, use `prores_4444_mov`.
