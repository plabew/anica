# TOOL-0003: Timeline Analysis and Edit Plan Tools

## Tools
- `anica.timeline/get_snapshot`
- `anica.timeline/get_audio_silence_map`
- `anica.timeline/build_audio_silence_cut_plan`
- `anica.timeline/get_subtitle_gap_map`
- `anica.timeline/build_subtitle_gap_cut_plan`
- `anica.timeline/validate_edit_plan`
- `anica.timeline/apply_edit_plan`

## Purpose
Analyze timeline structure and apply safe edits through validated edit plans.

## B-roll Placement Principles
- The best B-roll insertion points usually fall into three cases:
  - When the speaker mentions concrete objects: add visuals that directly support what is being discussed.
  - During stumbles or repeated phrases: use B-roll to cover cut points so jump cuts feel less abrupt.
  - When abstract ideas need visualization: use imagery to make the concept easier to understand.

## Deprecation Note
- `anica.timeline/get_subtitle_semantic_repeats` is disabled.
- Similar/repeated sentence detection must use the LLM-only flow.

## Effect Naming Note
- For clip color overlay effect naming, use `hsla_overlay` as the canonical name.
- Legacy alias `tint` is still accepted for backward compatibility, but ACP replies/prompts should prefer `hsla_overlay`.

## Workflow Rule (Mandatory)
1. Read timeline state (`get_snapshot` / analysis tool).
2. Build plan.
3. Validate with `validate_edit_plan`.
4. Apply with `apply_edit_plan`.

- Before `anica.timeline/validate_edit_plan`, you must call `anica.timeline/get_snapshot` first.
- Use the returned `timeline_revision` as `based_on_revision`.
- Do not use placeholders like `rev_xxx`.
- If timeline changes, fetch snapshot again and re-validate.

### Strategy Isolation Rule (Mandatory)
- If user intent is **audio silence cut** (`silence`, `quiet`, `靜音`, `空白聲`):
  - Use `get_audio_silence_map` / `build_audio_silence_cut_plan`.
  - **Do not fallback** to subtitle-gap tools.
- If user intent is **subtitle-gap cut**:
  - Use `get_subtitle_gap_map` / `build_subtitle_gap_cut_plan`.
  - **Do not fallback** to audio-silence tools.

## Operation Schema Quick Reference (v1)

### 1) `add_track`
- Required:
  - `op`: `"add_track"`
  - `track_type`: `"audio"` | `"video"` | `"subtitle"`
- Optional:
  - `name`

### 2) `insert_from_media_pool` (recommended for AI B-roll)
- Required:
  - `op`: `"insert_from_media_pool"`
  - `track_type`: `"v1"` | `"audio"` | `"video"`
  - `media_pool_item_id`
- Optional:
  - `track_index` (required when `track_type` is `audio` or `video`)
  - `start_ms`
  - `source_in_ms`
  - `source_out_ms`
  - `duration_ms`

### 3) `insert_clip` (path fallback)
- Required:
  - `op`: `"insert_clip"`
  - `track_type`: `"v1"` | `"audio"` | `"video"`
  - `path` or `media_pool_item_id`
- Optional:
  - `track_index` (required when `track_type` is `audio` or `video`)
  - `start_ms`
  - `source_in_ms`
  - `source_out_ms`
  - `duration_ms`

### 4) `set_source_in_out`
- Required:
  - `op`: `"set_source_in_out"`
  - `clip_id`
  - `source_in_ms`
- Optional:
  - `source_out_ms`
  - `duration_ms`

`set_source_in_out` behavior:
- If `source_out_ms` is provided, it wins over `duration_ms`.
- If only `source_in_ms` is provided, duration uses remaining media length.

### 5) `delete_track_clips`
- Required:
  - `op`: `"delete_track_clips"`
  - `track_type`: `"v1"` | `"audio"` | `"video"` | `"subtitle"`
- Optional:
  - `track_index` (required for `audio` / `video` / `subtitle`; for `v1` use `0` or omit)
  - `with_linked` (default: `true`; when deleting video/audio clips, also delete linked A/V companions)

### 6) `insert_semantic_clip` (B-roll planning marker)
- Required:
  - `op`: `"insert_semantic_clip"`
  - `start_ms`
  - `duration_ms`
- Optional:
  - `semantic_type` (recommended, e.g. `內容補充` / `遮蓋剪接`)
  - `label` (short shot note)

```json
    {
      "op": "delete_track_clips",
      "track_type": "video",
      "track_index": 1,

  ]

```

### 2b) Insert B-roll from Media Pool with source window (recommended)
```json
{
  "based_on_revision": "rev_from_latest_snapshot",
  "operations": [
    {
      "op": "insert_from_media_pool",
      "track_type": "video",
      "track_index": 0,
      "media_pool_item_id": 2,
      "start_ms": 15000,
      "source_in_ms": 1200,
      "source_out_ms": 4200
    }
  ]
}
```

### 2c) Update an existing clip source window
```json
{
  "based_on_revision": "rev_from_latest_snapshot",
  "operations": [
    {
      "op": "set_source_in_out",
      "clip_id": 42,
      "source_in_ms": 1200,
      "source_out_ms": 4200
    }
  ]
}
```

`insert_clip` source selection rules:
- Use `media_pool_item_id` (recommended) to avoid path mismatch.
- `path` is still supported as fallback.
- `source_out_ms` takes precedence over `duration_ms`.
- If only `source_in_ms` is provided, duration uses `duration_ms` or remaining media length.

## Common Failures
- `Revision mismatch`: `based_on_revision` is stale or placeholder.
- `No operations provided`: `operations` is empty or missing.

## Notes
- Do not mutate timeline directly without validate/apply sequence.
- `Broll suggestion` should be treated as execute intent for semantic markers: build aggressive `insert_semantic_clip` ops, then validate and apply directly unless user explicitly asks validate-only.
- Trigger language variants:
  - `Broll suggestion (use <language> write suggestion)` => detect requested language and write marker labels in that language.
  - `Broll suggestion` (default) => infer dominant subtitle language from snapshot and use that language.
- `Broll suggestion` flow is append-only: add new semantic markers and never replace/delete existing semantic markers.
- `atomic` defaults to `true`. If any op fails, apply stops immediately.
- For subtitle gap mode default, use `balanced` unless user requests otherwise.
- For semantic-repeat defaults: `window_ms=30000`, `similarity_threshold=0.90`.
