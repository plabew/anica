# TOOL-0005: MotionLoom VFX Tools

## Tools
- `anica.motionloom/get_scene_script`
- `anica.motionloom/set_scene_script`
- `anica.motionloom/render_scene`

## Purpose
Control MotionLoom VFX page from ACP:
- read current DSL,
- update DSL (with parser/runtime validation),
- trigger render actions equivalent to VFX page buttons.

## Render Mode Mapping

`anica.motionloom/render_scene` modes:
- `gpu` -> VFX button: `GPU Render`
- `gpu_prores` -> VFX button: `GPU Render (ProRes)`
- `compatibility_cpu` -> VFX button: `Compatibility Render (CPU)`

## Workflow (Recommended)
1. Generate valid `<Graph ...>` script.
2. Call `set_scene_script` with:
   - `apply_now=true`
   - `focus_vfx_page=true`
3. If user wants output media:
   - call `render_scene` with target mode.

## Validation Behavior

`set_scene_script` validates before accept:
- must be Graph DSL (`<Graph ...>...</Graph>`)
- parse must pass
- compile must pass
- unsupported kernels are rejected
- auto-defaults if missing on `<Graph ...>`:
  - `fps={60}`
  - `size={[1920,1080]}`
- compatibility normalization:
  - if `<Graph>` has no `duration` and `<Present>` has `duration_ms`, duration is auto-filled into `<Graph duration="...">`
  - `<Text text|content|children|label="...">` and `<Text>...</Text>` are rewritten to `<Text value="..." />`
  - simple `<Animate target="id.opacity" ... />` is rewritten to node `opacity="..."` expression

If invalid (for example `kind="raster"`), tool returns error and script is not committed.

## Request Shapes

### 1) Read script
```json
{}
```

### 2) Set script
```json
{
  "script": "<Graph scope=\"scene\" fps={60} duration=\"3s\" size={[1920,1080]}>...</Graph>",
  "apply_now": true,
  "focus_vfx_page": true
}
```

### 3) Render scene
```json
{
  "mode": "gpu",
  "focus_vfx_page": true
}
```
