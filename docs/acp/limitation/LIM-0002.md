# LIM-0002: 8K Export May Not Use VideoToolbox GPU Path

## Status
- State: `open`
- Priority: `medium`
- Type: export capability limitation

## Summary
On macOS, `h264_videotoolbox` may fail at 8K (`7680x4320`) depending on hardware/session limits.
When this happens, Anica automatically falls back to a CPU export path so the export can still finish.

## Scope
- Platform: macOS
- Preset: `h264_videotoolbox_mp4`
- Stage: export (not preview)

## Resolution Behavior (Observed)
- `1920x1080`: VideoToolbox success
- `3840x2160`: VideoToolbox success
- `7680x4320`: VideoToolbox failure, automatic CPU fallback

## Why It Happens
VideoToolbox hardware sessions have device-specific limits for codec profile/level and frame size.
At 8K, encoder session creation can fail on some systems.

## User Impact
- Export is not blocked.
- Export continues via CPU fallback path.
- Resulting file is still produced successfully.

## ACP Reply (Short)
8K export may not stay on the VideoToolbox GPU path on macOS. If VideoToolbox cannot create the encoder session, Anica automatically falls back to CPU export so the job can complete.

## ACP Reply (Detailed)
Your export behavior is expected. On some macOS hardware, `h264_videotoolbox` works at 1080p and 4K but fails at 8K (`7680x4320`) due to hardware encoder limits.  
When this happens, Anica automatically switches to CPU fallback and continues export.  
This is a known limitation of the GPU encoder path at very high resolutions, not project corruption.
