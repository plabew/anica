# Trademark And Branding

This policy covers branding only (project name, logo, release identity).
Code usage permissions are defined by `LICENSE`.

## Not Permitted Without Written Permission
- Present non-maintainer binaries as official Anica releases.
- Use the Anica name/logo in a way that implies endorsement or partnership.
- Reuse the official bundle identifier for redistributed binaries.

## Branding Requirements For Third-Party Distribution
- Use a different app display name.
- Use a different bundle identifier.
- Use your own icon and branding assets.
- Clearly mark the build as unofficial.

## Official Releases
Only releases published by the Anica maintainers are official.
