# Security Policy

If you believe you found a security vulnerability, please prefer private reporting first.
If private reporting is not available, open a public issue without exploit details.


## How to Report

1. Use GitHub Security Advisories (private reporting) when available.
2. If private advisories are not available, contact maintainers through
   repository contact channels and label the message `SECURITY`.

Helpful details:

- affected version or commit
- reproduction steps or proof of concept
- expected impact

## Policy Notes

- Reports are handled on a best-effort basis.
- We prefer coordinated disclosure and ask that you avoid public disclosure
  before maintainers have time to investigate and prepare a fix.
